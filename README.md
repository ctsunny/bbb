## 📋 完整方案书

### 一、项目概述

#### 1.1 项目定位
NanoMonitor 是一款专为 Linux VPS 设计的轻量级网页内容变动监控工具。它通过无头浏览器自动抓取指定网页，利用智能算法过滤广告、时间戳等噪音内容，精准识别页面核心内容的变化，并通过 Bark 推送实时通知用户。

#### 1.2 解决痛点
- **手动刷新繁琐**：无需人工定期检查网页更新
- **信息遗漏风险**：7×24 小时自动监控，不错过任何变动
- **噪音干扰严重**：智能过滤广告、推荐位、时间等非核心内容
- **部署复杂**：传统监控工具配置繁琐，本工具一行命令即可使用
- **资源占用高**：优化浏览器实例管理，低配 VPS 也能流畅运行

#### 1.3 适用场景
- 📰 新闻网站更新监控（政策发布、行业新闻）
- 💰 价格变动追踪（电商商品、机票酒店）
- 📢 公告通知监测（学校通知、公司通告）
- 📝 博客文章更新（关注博主新文章）
- 🔒 安全漏洞公告（CVE、厂商安全通告）
- 📊 数据页面监控（汇率、股票、排行榜）

### 二、核心功能

#### 2.1 智能网页抓取
- **无头浏览器**：基于 Puppeteer 的 Chromium，支持 JavaScript 渲染
- **完整页面加载**：等待动态内容加载完成再截图/提取
- **自定义选择器**：可指定监控页面的特定区域（如 `.content`、`#news-list`）
- **请求头定制**：支持设置 User-Agent、Cookie 等模拟真实访问

#### 2.2 智能噪音过滤
- **广告过滤**：自动识别并移除常见广告容器
- **时间戳忽略**：识别日期格式内容，避免因时间变化误报
- **随机内容过滤**：过滤随机 ID、会话标识等动态内容
- **白名单机制**：可设置只监控特定元素，忽略其他所有内容

#### 2.3 精准差异对比
- **文本 diff 算法**：精确比对两次抓取的文本差异
- **可视化对比**：Web 界面高亮显示新增/删除/修改的内容
- **差异摘要**：自动生成变更摘要，快速了解变动概况
- **历史记录**：保存每次抓取的快照和对比结果

#### 2.4 灵活调度系统
- **Cron 表达式**：支持标准 cron 语法，精确控制执行时间
- **预设模板**：提供常用间隔（每 5 分钟、每小时、每天等）
- **防重叠执行**：任务超时未完成时，下次执行自动跳过
- **时区支持**：可设置任务执行的时区

#### 2.5 实时通知推送
- **Bark 推送**：iOS 用户可通过 Bark APP 接收即时通知
- **推送内容**：包含变动摘要和详情链接
- **静默时段**：可设置免打扰时间段
- **推送频率限制**：避免短时间内重复推送

#### 2.6 Web 管理面板
- **任务管理**：添加、编辑、删除、启用/禁用监控任务
- **实时监控**：查看任务执行状态、上次执行时间、下次执行时间
- **历史记录**：浏览历史抓取记录和差异对比
- **系统设置**：修改端口、账号密码、访问路径
- **响应式设计**：支持手机、平板、电脑访问

### 四、使用说明

#### 4.1 快速开始（3 步上手）

**步骤 1：一键安装**
```bash
curl -fsSL https://raw.githubusercontent.com/ctsunny/bbb/main/install.sh | sudo bash
```

**步骤 2：获取登录信息**
安装完成后终端会显示：
```
📍 访问地址：http://1.2.3.4:3000/a1b2c3d4
👤 管理员账号：admin
🔑 管理员密码：Ab3xK9mNpQr2
```

**步骤 3：添加监控任务**
1. 浏览器访问上述地址
2. 使用账号密码登录
3. 点击「添加任务」
4. 填写监控 URL、选择器、执行间隔
5. 保存后系统自动开始监控

#### 4.2 创建监控任务详解

**必填参数：**
- **任务名称**：自定义标识，如「GitHub Trending 监控」
- **监控 URL**：要监控的网页地址，如 `https://github.com/trending`
- **执行间隔**：使用 cron 表达式或预设模板

**可选参数：**
- **CSS 选择器**：只监控特定区域，如 `.Box-body`（留空则监控全文）
- **噪音规则**：选择要过滤的元素类型（广告、时间戳等）
- **Bark 推送 Key**：填写后变动时会推送通知
- **启用状态**：可临时禁用任务而不删除

**Cron 表达式示例：**
| 表达式 | 含义 |
|--------|------|
| `*/5 * * * *` | 每 5 分钟执行一次 |
| `0 * * * *` | 每小时整点执行 |
| `0 9 * * *` | 每天早上 9 点执行 |
| `0 0 * * 1` | 每周一凌晨执行 |
| `0 */2 * * *` | 每 2 小时执行一次 |

#### 4.3 查看监控结果

**任务列表页：**
- 显示所有任务的名称、URL、状态、上次执行时间、下次执行时间
- 绿色表示正常，红色表示执行失败
- 可快速启用/禁用、编辑、删除任务

**历史记录页：**
- 按时间倒序展示每次执行结果
- 点击任意记录查看详细内容
- 支持关键词搜索历史记录

**差异对比页：**
- 左侧显示旧内容，右侧显示新内容
- 红色背景表示删除的内容
- 绿色背景表示新增的内容
- 黄色高亮表示修改的内容
- 顶部显示差异统计（新增 X 行，删除 Y 行）

#### 4.4 管理命令

```bash
nanomon  # 打开管理菜单
```

**菜单选项说明：**

| 选项 | 功能 | 说明 |
|------|------|------|
| 1 | 一键升级 | 备份数据 → 拉取代码 → 恢复配置 → 重启服务 |
| 2 | 重启服务 | 立即重启 systemd 服务 |
| 3 | 停止服务 | 停止监控任务，释放资源 |
| 4 | 启动服务 | 重新启动监控服务 |
| 5 | 查看日志 | 实时滚动显示最新日志 |
| 6 | 重置密码 | 生成新的随机管理员密码 |
| 7 | 完全卸载 | 删除所有程序文件和数据 |

### 六、安全最佳实践

#### 6.1 访问安全
- ✅ 默认生成随机访问路径（如 `/a1b2c3d4`），防止暴力扫描
- ✅ 强制 HTTPS（需自行配置反向代理）
- ✅ 定期更换管理员密码
- ✅ 限制 IP 访问（通过防火墙或 Nginx）

#### 6.2 网络安全
```bash
# Ubuntu/Debian 配置 UFW 防火墙
sudo ufw allow 3000/tcp
sudo ufw limit 3000/tcp  # 限制连接频率
sudo ufw enable

# CentOS 配置 firewalld
sudo firewall-cmd --add-port=3000/tcp --permanent
sudo firewall-cmd --reload
```

#### 6.3 数据安全
- ✅ 数据库自动备份（升级时自动执行）
- ✅ 配置文件加密存储密码
- ✅ 定期手动备份 `/var/lib/nanomon/` 目录

**手动备份命令：**
```bash
sudo tar -czf nanomon_backup_$(date +%Y%m%d).tar.gz /var/lib/nanomon/
```

### 七、故障排查

#### 7.1 诊断工具

```bash
# 检查服务状态
sudo systemctl status nanomon

# 查看详细日志
sudo journalctl -u nanomon -n 100 --no-pager

# 实时跟踪日志
sudo journalctl -u nanomon -f

# 检查端口占用
sudo netstat -tlnp | grep 3000

# 检查进程
ps aux | grep nanomon

# 检查数据库
sqlite3 /var/lib/nanomon/nanomon.db ".tables"
sqlite3 /var/lib/nanomon/nanomon.db "SELECT * FROM tasks;"
```

#### 7.2 常见问题速查表

| 问题现象 | 可能原因 | 解决方案 |
|----------|----------|----------|
| 服务无法启动 | 端口被占用 | `netstat -tlnp \| grep 3000` 查找并关闭占用进程 |
| 页面空白 | 前端文件缺失 | 重新运行安装脚本修复 |
| 任务不执行 | Cron 表达式错误 | 检查 cron 语法，使用预设模板 |
| 推送失败 | Bark Key 错误 | 验证 Bark Key 是否正确 |
| 内存过高 | 浏览器未释放 | 升级到 v1.7.4+（已修复实例复用） |
| 抓取失败 | 目标网站反爬 | 设置 User-Agent，添加延迟 |
| 差异过多 | 噪音未过滤 | 配置 CSS 选择器限定监控区域 |

## 🔧 常见问题

### 1. 服务无法启动

```bash
# 查看服务状态
sudo systemctl status nanomon

# 查看详细日志
sudo journalctl -u nanomon -n 50

# 尝试重启
sudo systemctl restart nanomon
```

### 2. Puppeteer 浏览器无法启动

```bash
# 安装缺失的依赖
sudo apt-get update
sudo apt-get install -y \
    libgbm1 libasound2 libatk1.0-0 libatk-bridge2.0-0 \
    libcups2 libdbus-1-3 libdrm2 libgtk-3-0 libnspr4 \
    libnss3 libxkbcommon0 libxrandr2 xdg-utils \
    fonts-liberation libappindicator3-1
```

### 3. 内存占用过高

NanoMonitor 已优化浏览器实例复用，正常情况下内存占用约 200-400MB。如仍过高：

```bash
# 查看内存使用
ps aux | grep nanomon | grep -v grep

# 重启服务释放内存
sudo systemctl restart nanomon
```

### 4. 监控任务不执行

```bash
# 检查日志
sudo journalctl -u nanomon -f

# 检查数据库
sqlite3 /var/lib/nanomon/nanomon.db "SELECT * FROM tasks;"
```
